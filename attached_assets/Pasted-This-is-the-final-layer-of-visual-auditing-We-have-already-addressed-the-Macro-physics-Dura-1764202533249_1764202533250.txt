### **I. The "Soft" Leaks & The "Hard" Fixes**

#### **1. The Text Selection (The "Highlighter" Effect)**

  * **The Issue:** Default browser text selection is usually a bright blue. It feels generic.
  * **The Institutional Fix:** When a user selects text on your site, it should look like they are highlighting data in a terminal.
  * **The Code:** Add this to `client/src/index.css`:
    ```css
    ::selection {
      background-color: hsl(var(--primary) / 0.2); /* Low opacity brand color */
      color: hsl(var(--foreground));
      -webkit-text-fill-color: hsl(var(--foreground));
    }
    ```

#### **2. The Scrollbar (The "Browser" Residue)**

  * **The Issue:** A standard gray/white browser scrollbar breaks the "Dark Mode" immersion. It reminds the user they are on a website.
  * **The Institutional Fix:** A thin, dark, almost invisible scrollbar that only appears when needed.
  * **The Code:** Add to `client/src/index.css`:
    ```css
    ::-webkit-scrollbar {
      width: 10px;
      background: hsl(var(--background));
    }
    ::-webkit-scrollbar-thumb {
      background: hsl(var(--border));
      border-radius: 0px; /* Square corners = Engineering */
      border: 2px solid hsl(var(--background)); /* Creates padding effect */
    }
    ::-webkit-scrollbar-thumb:hover {
      background: hsl(var(--muted-foreground));
    }
    ```

#### **3. The "Toast" Notification Physics**

  * **The Issue:** Shadcn/Radix toasts often have a "swipe" animation or a "bounce" when they appear.
  * **The Institutional Fix:** Toasts should strictly **Fade In** and **Fade Out**. No sliding from the side.
  * **Action:** Your `client/src/components/ui/toast.tsx` uses `slide-in-from-top-full`.
      * **Change:** Replace all `slide-in-...` and `slide-out-...` classes with `fade-in-0` and `fade-out-0`.

#### **4. The "Focus" Ring (The Glow)**

  * **The Issue:** When you tab-select an input or button, the default Tailwind ring is a soft, fuzzy glow (`ring-2 ring-offset-2`).
  * **The Institutional Fix:** A sharp, 1px hard border.
  * **The Logic:** Engineering tools use precision, not diffusion.
  * **Action:** In `client/src/index.css` or `tailwind.config.ts`, override the ring utility to be `0px` blur.
      * *Easier Fix:* In your global button/input components, ensure `focus-visible:ring-1` (thinner) and add `focus-visible:ring-offset-0` (tighter).
